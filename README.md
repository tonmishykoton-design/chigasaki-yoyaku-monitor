# 茅ヶ崎市施設予約 空き状況モニター

茅ヶ崎市公共施設予約サービスの「空き状況の確認」を毎日朝9時・夜11時に自動チェックし、
対象施設・対象曜日・対象時間帯に空きがあれば Gmail に通知します。

## 監視対象(初期設定)

- 茅ヶ崎市総合体育館 柔道場
- 茅ヶ崎市総合体育館 剣道場
- 茅ヶ崎市総合体育館 オーケストラ室
- 茅ヶ崎市体育館 柔剣道場
- 曜日: 毎週日曜日
- 時間帯: 12:00〜15:00
- 期間: サイトで検索可能な範囲まで自動で先の週へ進めてチェック

変更したい場合は `config.py` を編集してください。

## セットアップ手順

### 1. Gmailアプリパスワードを発行する

通常のGmailパスワードではSMTP送信できないため、アプリパスワードが必要です。

1. Googleアカウントの[セキュリティ設定](https://myaccount.google.com/security)を開く
2. 「2段階認証プロセス」を有効にする(まだの場合)
3. 「アプリパスワード」を開き、任意の名前(例: chigasaki-yoyaku)で発行
4. 表示された16桁のパスワードをコピーしておく(この画面以降は再表示されません)

### 2. GitHubリポジトリにSecretsを登録する

このリポジトリの `Settings` → `Secrets and variables` → `Actions` → `New repository secret` から、以下の3つを登録してください。

| Secret名 | 値 |
|---|---|
| `GMAIL_USER` | 送信に使うGmailアドレス |
| `GMAIL_APP_PASSWORD` | 手順1で発行したアプリパスワード(16桁) |
| `NOTIFY_TO` | 通知を受け取りたいメールアドレス(例: tonmishykoton@gmail.com) |

### 3. ファイルを配置する

このフォルダの中身をそのままリポジトリのルートに置いてください。

```
リポジトリ直下/
├── check_availability.py
├── config.py
├── requirements.txt
└── .github/workflows/check.yml
```

### 4. 動作確認

コミット・pushした後、GitHubリポジトリの `Actions` タブ →
「茅ヶ崎市施設予約 空き状況チェック」→ 右上の `Run workflow` から
手動実行して、正常に動くか確認してください。
問題なければ、あとは自動で毎日9時・23時に実行されます。

## 注意点・既知の制約

- 予約サイトは**古いフレーム構成のシステム**のため、リンクの表示文言をもとに
  画面遷移を自動操作しています。サイト側で文言やレイアウトが変わった場合は
  `check_availability.py` 内のリンクテキスト(`"空き状況の確認"` など)を
  実際の画面に合わせて調整してください。
- 実際のサイトに対する自動テストができない環境で作成したため、
  **初回実行時にセレクタの微調整が必要になる可能性があります**。
  `Actions` タブの実行ログでエラー箇所を確認しながら調整してください。
- サイトへのアクセス頻度は1日2回に抑えてあります。手動実行を含め、
  短時間に何度も実行しないようご配慮ください。
- GitHub Actionsの無料枠(パブリックリポジトリは無制限、プライベートは月2,000分)で
  十分に収まる想定です。
